# WSL2をWindows Terminalで操作する

# Powerlineフォント対応

リガチャとか合字って開発で意識したことなかったけど解説 (opens new window)まであった。

chocoで管理する (opens new window)

白源 (opens new window)ってフォントも見つけて Powerline対応してるらしいけど、Windows Terminalだと表示されなかった。

とりあえずWindows TerminalとVSCodeのターミナルは「Cascadia Code PL」にしといた。

で、oh-my-zshの有名なPowerlineテーマBullet Train (opens new window)を入れた。 非公式テーマ (opens new window)は、$ZSH_CUSTOM/themesに置く。シンボリックでOK。

# WSL2からポートフォワード

.wslconfigを設置するだけ。

Wslconfig を使用してグローバルオプションを構成する (opens new window)

vmはポートフォワードをvm毎に書いてるけど、 そういう手間なくlocalhost:8080などでwindowsからwsl2にアクセスできるようになる。 でも、まだ不安定みたい (opens new window)で、 自分の環境でも症状っぽいのは確認。

# WSL2(ホスト)からコンテナアクセス

mysql (opens new window)を例にすると、 WSL2(ホスト)からmysql(コンテナ)へlocalhostではアクセスできないが::1なら接続できた。 docker run -i (--interactive)しないとエラーメッセージが出力されなくて少しハマった。
